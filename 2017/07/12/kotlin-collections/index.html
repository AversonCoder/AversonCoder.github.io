集合懒加载

使用该操作的结果和前面是一样的，但是区别在于后者并没有在过程中创建两个集合。在 Kotlin 中 Sequence 接口是延迟收集元素操作的关键。该接口表示列举所有元素的一个序列。该接口仅提供一个方法 iterator()，你可以使用迭代器从序列中获取值。该接口的优势在于它实现的操作方法。序列中的元素均被延迟计算。因此，你可以使用序列来高效执行对集合的操作，而不需要存储中间的操作结果集。

你能通过集合的拓展函数（asSequence()）把集合转换为序列。调用 toList() 进行反向转换。为什么需要把序列转换回集合？这样不是更方便？然而答案是：有时。如果你需要遍历序列中的所有元素，就可以直接使用序列。如果你需要使用其他 API 的方法，如按照索引访问元素，那么就需要把序列转换为集合(List)了。

一个好的建议：当在大型集合上有链式操作的时候，使用序列化。

在序列上操作分为过渡操作和终止操作两种类型。过渡操作返回另外一个序列，表明如何转换原始序列。而终止操作返回一个结果，可能是集合、元素、数字和其他对象。

序列的整个链式操作过程中如果没有出现终止符，那么就不会执行操作。

另外操作符顺序对集合性能也有影响。

Java 8 Stream 和 Kotlin Sequences 基本类似，之所以 Kotlin 重新实现是因为平台的问题，比方说 Android 只有 API 大于等于 23 才能使用 Java 8。并且 Java 8 Stream 支持并行。

Kotlin 除了可以把集合转换为序列以外，还提供了相关的 API 生成序列。

  1. 提供一个生成规则（定义一个初始值为 0 并每次加一且加一后的结果作为下一次的初始值的规则）
  2. 提供一个有限范围条件（当生成的值小于等于 100 一直获取并存为一个集合）
  3. 使用终止操作符开始执行
